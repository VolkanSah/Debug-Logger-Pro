How It Works

The plugin hooks into the WordPress API to capture and store all critical events and errors in a dedicated log file. For the most comprehensive error capture, it integrates directly with the WP_DEBUG constant.

wp-config.php Configuration 
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);

The plugin will then read this standard WordPress log file and display its content directly in the UI.

1. Error Capture

Hooks into WordPress error handling systems to catch all errors and warnings.

2. Log Storage

Stores errors in a dedicated log file with timestamps and context.

3. Real-time Display

Presents the log data in an easy-to-use interface with filtering options.

Getting Started (Quick Guide)

1 Download & Install

  1. i Download the plugin archive from the GitHub repository.
  2. ii In your WordPress admin, go to Plugins > Add New > Upload Plugin and select the file.
  3. iii Activate the plugin.

2 Configure & Use

  1. i Navigate to Tools > Debug Logger Pro to enable the console.
  2. ii Configure your settings (optional).
  3. iii The debug console will appear on your admin pages.
